
## Voice Server Integration - Webchat Manual Posting

**Decision:** All webchat responses must be manually posted to voice server via curl.

**Context:** Auto-voice hooks (voice-streaming, voice-auto-speak) were causing:
- Chunked/incomplete playback (only first line spoken)
- Duplicate playback (TTS patch + webchat client both posting)

**Solution:**
1. Disabled all auto-voice hooks in gateway config
2. Disabled TTS patch `postToVoiceServer()` function
3. Disabled webchat client auto-speak JavaScript
4. Added mandatory instructions to AGENTS.md for manual posting
5. Updated TOOLS.md with technical details

**Command:** 
```bash
curl -s -X POST http://127.0.0.1:18790/speak \
  -H "Content-Type: application/json" \
  -d '{"text":"RESPONSE_TEXT", "immediate": true}'
```

**Tested:** 2026-02-07 09:11 EST - Works perfectly, no duplicates

**Files modified:**
- AGENTS.md (added Webchat Voice Integration section)
- TOOLS.md (updated voice server notes)
- ~/.openclaw/TTS-VOICE-PATCH.md (documented manual approach)
- dist/tts/tts.js (disabled postToVoiceServer)
- dist/control-ui/index.html (disabled auto-speak JS)
- config hooks disabled: voice-streaming, voice-auto-speak
